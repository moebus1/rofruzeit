FruchtsaftZeit einrichten
Einmalige Ersteinrichtung mit Supabase
Schritt 1 – Supabase Projekt
Erstelle ein kostenloses Konto auf supabase.com, lege ein neues Projekt an und kopiere unter Project Settings → API die Project URL und den anon key.
Schritt 2 – Datenbank einrichten
Führe im SQL-Editor deines Supabase-Projekts folgenden Code aus:
-- Profil-Tabelle (ergänzt Supabase Auth) CREATE TABLE profiles ( id uuid REFERENCES auth.users PRIMARY KEY, name text NOT NULL, role text NOT NULL DEFAULT 'employee', department text ); -- Schichten CREATE TABLE shifts ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, employee_id uuid REFERENCES profiles(id), date date NOT NULL, start_time time NOT NULL, end_time time NOT NULL, shift_type text DEFAULT 'Frühschicht', note text ); -- Zeitbuchungen CREATE TABLE time_entries ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, employee_id uuid REFERENCES profiles(id), date date NOT NULL, clock_in time NOT NULL, clock_out time, duration_minutes integer, note text ); -- Verfügbarkeiten CREATE TABLE availabilities ( id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY, employee_id uuid REFERENCES profiles(id) UNIQUE, days text[] DEFAULT '{}' ); -- Row Level Security ALTER TABLE profiles ENABLE ROW LEVEL SECURITY; ALTER TABLE shifts ENABLE ROW LEVEL SECURITY; ALTER TABLE time_entries ENABLE ROW LEVEL SECURITY; ALTER TABLE availabilities ENABLE ROW LEVEL SECURITY; -- Policies: Admin/Planner sehen alles, Employee nur eigenes CREATE POLICY "read_own_or_staff" ON profiles FOR SELECT USING ( id = auth.uid() OR (SELECT role FROM profiles WHERE id = auth.uid()) IN ('admin','planner') ); CREATE POLICY "all_shifts_staff" ON shifts FOR ALL USING ( (SELECT role FROM profiles WHERE id = auth.uid()) IN ('admin','planner') ); CREATE POLICY "own_shifts_employee" ON shifts FOR SELECT USING ( employee_id = auth.uid() ); CREATE POLICY "own_entries" ON time_entries FOR ALL USING ( employee_id = auth.uid() OR (SELECT role FROM profiles WHERE id = auth.uid()) IN ('admin','planner') ); CREATE POLICY "own_avail" ON availabilities FOR ALL USING ( employee_id = auth.uid() OR (SELECT role FROM profiles WHERE id = auth.uid()) IN ('admin','planner') );
